VS Code Settings

Create .vscode/settings.json in your project root:

{
    "python.defaultInterpreterPath": "./backend/.venv/bin/python",
    "python.formatting.provider": "black",
    "python.linting.enabled": true,
    "python.linting.pylintEnabled": true,
    "files.associations": {
        "*.html": "html",
        "*.po": "gettext"
    },
    "emmet.includeLanguages": {
        "django-html": "html"
    },
    "[python]": {
        "editor.formatOnSave": true,
        "editor.codeActionsOnSave": {
            "source.organizeImports": true
        }
    }
}

Code Quality Tools

Formatting and Linting

Install development tools in your Poetry environment:

cd backend
poetry add --group dev black isort pylint pytest pytest-django

Pre-commit Hooks

Set up pre-commit hooks to maintain code quality:

# Install pre-commit
poetry add --group dev pre-commit

# Install hooks
poetry run pre-commit install

Create .pre-commit-config.yaml:

repos:
  - repo: https://github.com/psf/black
    rev: 23.3.0
    hooks:
      - id: black
        language_version: python3.8

  - repo: https://github.com/pycqa/isort
    rev: 5.12.0
    hooks:
      - id: isort
        args: ["--profile", "black"]

  - repo: https://github.com/pycqa/flake8
    rev: 6.0.0
    hooks:
      - id: flake8
        args: ['--max-line-length=88', '--extend-ignore=E203']

  - repo: https://github.com/pre-commit/mirrors-eslint
    rev: v8.40.0
    hooks:
      - id: eslint
        files: \.(js|jsx|ts|tsx)$
        types: [file]

Database Setup

PostgreSQL Configuration

Ubuntu/Debian

# Install PostgreSQL
sudo apt update
sudo apt install postgresql postgresql-contrib

# Create database and user
sudo -u postgres psql
CREATE DATABASE linguify_db;
CREATE USER linguify_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE linguify_db TO linguify_user;
\q

macOS

# Install via Homebrew
brew install postgresql
brew services start postgresql

# Create database
createdb linguify_db

Windows

Download PostgreSQL installer from postgresql.org and use pgAdmin to create the database.

Environment Variables

Create backend/.env file:

# Database
DATABASE_URL=postgresql://linguify_user:your_password@localhost/linguify_db

# Django
SECRET_KEY=your-secret-key-here
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1

# Optional: Email configuration for development
EMAIL_BACKEND=django.core.mail.backends.console.EmailBackend

Development Workflow

Daily Workflow

  1. Activate virtual environment: poetry shell
  2. Pull latest changes: git pull origin develop
  3. Update dependencies: poetry install
  4. Run migrations: python manage.py migrate
  5. Start development servers
  6. Run tests before committing: pytest

Troubleshooting

Common Issues

If you have multiple Python versions:

  1. Use pyenv to manage Python versions
  2. Ensure Poetry uses the correct Python: poetry env use python3.8
  3. Check active version: poetry run python --version

If Poetry can't resolve dependencies:

  1. Clear Poetry cache: poetry cache clear pypi --all
  2. Remove lock file: rm poetry.lock
  3. Reinstall: poetry install
  4. Use poetry update to update all packages

If your IDE doesn't recognize the virtual environment:

  1. Find venv path: poetry env info --path
  2. Set interpreter in IDE to: [path]/bin/python
  3. Restart IDE after changing interpreter
  4. Ensure workspace is opened at project root
